# Contributing

Thanks for your interest in FlipbookEngine.

## Development Flow

1. Install dependencies with `npm install`
2. Run local development with `npm run dev`
3. Run type checks with `npm run typecheck`
4. Keep changes focused and documented

## Contribution Scope

Good first contribution areas:

- bug fixes
- performance improvements
- documentation
- test coverage
- framework integration samples

## Before Opening a PR

- explain the problem clearly
- keep the change set narrow
- avoid unrelated formatting churn
- update docs if behavior changes

To certify your agreement, you must sign off your git commits by adding a `Signed-off-by` line to every commit message (e.g., using `git commit -s`).

```

```

By signing your commits, you agree to the DCO version 1.1, whose full text is available at [developercertificate.org](https://developercertificate.org/).
